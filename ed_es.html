Ed - Un editor de texto orientado a líneas

[ English | Español ]

Introducción

GNU ed es un editor de texto orientado a líneas. Se utiliza para crear, visualizar, modificar y manipular ficheros de texto, tanto de forma interactiva como mediante scripts del intérprete de comandos (shell). Una versión restringida de ed, red, sólo puede editar archivos en el directorio actual y no puede ejecutar comandos del shell. Ed es el editor de texto "estándar" en el sentido de que es el editor original de Unix, y, por tanto, ampliamente disponible. Para la mayoría de los propósitos, sin embargo, es superado por los editores de pantalla completa como GNU Emacs o GNU Moe.

Documentación

El manual está disponible en el sistema info del Sistema Operativo GNU. Emplea el comando info para acceder al directorio del sistema info. Emplea info ed para acceder directamente a la sección de ed.

Se puede acceder al manual de ed desde aquí.

Descargar (Download) ed

La última versión publicada de GNU ed se puede encontrar en http://ftpmirror.gnu.org/ed/ o en el subdirectorio /gnu/ed/ en tu réplica GNU preferida. Para ver otras formas de obtener ed, consulta Cómo conseguir Software GNU. La última versión publicada será la versión más reciente disponible en http://ftp.gnu.org/gnu/ed/.

Puedes encontrar versiones antiguas y (si las hay) versiones beta para probar en http://download.savannah.gnu.org/releases/ed/.

Cómo Conseguir Ayuda

Para informar sobre errores (bugs) en ed, la lista de correo bug-ed@gnu.org es el lugar más adecuado. Por favor, envía los mensajes como texto llano. No envies HTML, base64 MIME ni múltiples formatos. Incluye una linea descriptiva en el asunto (subject) del mensaje. Si todos los asuntos son "error en ed" es imposible diferenciarlos.

Un archivo de la lista de correo está disponible en http://lists.gnu.org/mailman/listinfo/bug-ed.

Cómo Ayudar

Para contactar con el mantenedor, tanto para informar sobre un error (bug) como para contribuir soluciones o mejoras, envía un mensaje a bug-ed@gnu.org. Por favor, envía los mensajes como texto llano. Si envías parches, hazlo en formato diff unificado (diff -u) sobre la última versión. Incluye una descripción del problema en el mensaje.

También puedes visitar la página de ed en Savannah.
